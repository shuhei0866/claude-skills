# Codex コードレビュー

Codex CLI (`gpt-5.3-codex`) を使って、現在のブランチの変更を独立レビューします。
`/review-now`（Claude code-reviewer）と並列実行することで、2つの異なる AI による多角的レビューを実現します。

## 前提条件

- Codex CLI がインストール済みであること (`codex --version`)
- Codex にログイン済みであること (`codex login status` で確認)
  - ChatGPT ログイン (`Logged in using ChatGPT`) または API キー認証のいずれかでOK
- `~/.codex/config.toml` でモデルが設定済みであること
- 現在のブランチが `release/*` であること（推奨）

## 手順

### 1. 環境確認

```bash
codex --version
```

Codex CLI が見つからない場合はエラーを表示して終了する。

### 1.5. 認証確認

```bash
codex login status
```

ログイン済みであることを確認する。未ログインの場合:

```bash
codex login
```

ブラウザが開き、ChatGPT アカウントで認証できる。ブラウザが使えない環境では `codex login --device-auth` を使用する。

### 2. レビュー対象の決定

引数に応じてレビュー対象を決定する:

- 引数なし: main からの差分をレビュー (`--base main`)
- `--uncommitted`: 未コミットの変更をレビュー
- `--base <branch>`: 指定ブランチからの差分をレビュー
- `--pr <number>`: PR の差分をレビュー（`gh pr diff` → stdin）

$ARGUMENTS

### 3. Codex レビューの実行

```bash
# デフォルト: main からの差分
codex review --base main

# 未コミット変更の場合
codex review --uncommitted

# カスタムベースブランチ
codex review --base <branch>
```

コミットタイトルがある場合は `--title` で渡す:

```bash
codex review --base main --title "fix: 変更の概要"
```

**注意**: `--base` とプロンプト引数は同時に使えない（CLI の制約）。プロンプトを渡す場合は `--base` なしで使用する。

### 4. 結果の整形と出力

Codex の出力を以下のフォーマットに整形して表示する:

```markdown
## Codex Review Summary (gpt-5.3-codex)

### Critical Issues
- [ ] **[ファイル名:行番号]** 問題の説明

### Suggestions
- **[ファイル名:行番号]** 改善提案

### Good Points
- 良かった点

### Overall Assessment
総合評価と理由

---
*Reviewed by Codex CLI (gpt-5.3-codex)*
```

### 5. PR コメントへの投稿（オプション）

`--post` オプションが指定された場合、レビュー結果を PR にコメントとして投稿する:

```bash
gh pr comment --body "$(cat <<'EOF'
## Codex Review (gpt-5.3-codex)

[レビュー結果]

---
*This review was generated by Codex CLI (gpt-5.3-codex)*
EOF
)"
```

## `/review-now` との併用（RDD Phase 3）

RDD Phase 3 では `/review-now` と `/codex-review` を並列実行する:

```
/review-now       → Claude code-reviewer による独立レビュー
/codex-review     → Codex gpt-5.3-codex による独立レビュー
```

両方のレビューを通過してから PR を作成すること。
PR description には両方のレビュー結果サマリーを含めること。
